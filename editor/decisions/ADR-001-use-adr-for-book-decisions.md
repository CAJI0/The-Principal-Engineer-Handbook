# ADR-001: Use ADRs for Book Decisions

## Status

Accepted

## Context

The book project needs a durable way to document major editorial and architectural decisions.

## Decision

Use ADRs for major editorial and architectural decisions in the book project.

## Consequences

- Decisions have stable records.
- Future maintainers can understand why a direction was chosen.
- ADRs add lightweight process overhead.

## Alternatives Considered

- Track decisions only in issues.
- Track decisions only in commit messages.
