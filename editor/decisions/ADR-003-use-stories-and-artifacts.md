# ADR-003: Use Stories and Artifacts

## Status

Accepted

## Context

The book should avoid generic advice and make engineering thinking concrete.

## Decision

Each major idea should be supported by stories, artifacts, checklists, or decision records.

## Consequences

- Major ideas need concrete support.
- Editorial review must check whether abstract principles have context.
- Authors must provide or approve stories and artifacts.

## Alternatives Considered

- Present principles without supporting artifacts.
- Use code examples as the primary support for every idea.
