# Style Guide

This page mirrors the repository style guide.

> AUTHOR NOTE: Keep this page aligned with the canonical root `STYLE_GUIDE.md` file.

## Editorial Principles

- Preserve author meaning.
- Prefer small, reviewable edits.
- Use plain language where it does not alter the idea.
- Keep terminology consistent once authors define it.
- Flag uncertainty instead of resolving it by invention.

## Markdown

- Use one top-level `#` heading per page.
- Use sentence case for headings unless a proper noun requires otherwise.
- Keep one blank line around headings, lists, tables, and code blocks.
- Prefer reference links only when they improve maintainability.
- Use fenced code blocks with language identifiers.

## Cross-References

- Prefer relative links to other manuscript pages.
- Link to canonical definitions when a term is first used in a chapter.
- Do not create links to missing pages unless the target file is also created with an author note.
