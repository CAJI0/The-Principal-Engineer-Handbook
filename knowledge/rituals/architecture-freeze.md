# Architecture Freeze

## Purpose

Stabilize architectural decisions during a high-risk phase.

## Cadence

Use before validation, field trials, or release hardening.

## Participants

Technical leads, release owner, test owner, and owners of frozen boundaries.

## Process

Name frozen decisions, define allowed exceptions, and record exit criteria.

## Output

Freeze scope, exception log, and review date.

## Failure Modes

The freeze becomes permanent, vague, or used to avoid necessary fixes.
