# Weak Signal Register

## Purpose

Track early architecture concerns before they become confirmed failures.

## When To Use

Use it when patterns are visible but evidence is incomplete.

## Template

- Signal
- Where observed
- Possible cause
- Confidence
- Next evidence to gather
- Review date

## Common Mistakes

- Treating weak signals as proven defects.
- Recording vague discomfort without an observation.
- Never reviewing old entries.
