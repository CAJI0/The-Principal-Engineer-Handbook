# Failure Stories

Short failure-story records used to ground engineering concepts.

Failure story entries define concise evidence records, not full manuscript chapters. IDs use the `FAILURE-###` prefix and are assigned in `../index.yaml`.

Each concept file must include YAML front matter with `id`, `type`, `status`, and `canonical_name`.

Relationships are maintained in `../index.yaml`.
