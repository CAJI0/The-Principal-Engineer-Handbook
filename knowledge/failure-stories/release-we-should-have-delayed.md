# Release We Should Have Delayed

## Definition

A release shipped despite unresolved architectural risk.

## Why It Matters

Release pressure can convert known uncertainty into field cost.

## Symptoms

- Open risks lack owners.
- Test failures are explained away.
- Late architecture changes continue during hardening.

## Example

A device release proceeds with known update instability because failures are rare in lab conditions.
