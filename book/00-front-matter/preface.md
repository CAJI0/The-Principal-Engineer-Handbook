# Preface

This is not a book about C.

This is not a book about STM32.

This is not a book about RTOS APIs.

This is a book about engineering decisions, architecture, longevity, and technical leadership.

Embedded systems are the main context because they make architectural mistakes expensive. Hardware constraints, field failures, long product lifetimes, difficult updates, and real-world operating conditions punish vague ownership and casual design.

The reader should expect stories, ADRs, design reviews, failure stories, laws, checklists, and reference project material. Code appears only when it clarifies an architectural decision.

This book is meant for Senior, Staff, Principal, and Lead Embedded Engineers who are responsible for systems that must survive change.
